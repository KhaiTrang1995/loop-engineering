# Loop Design Docs

Generated by the `loop-design` skill (`templates/SKILL.md.loop-design`) — one
file per pattern, named `{pattern-slug}.md`, following
[`templates/loop-design-template.md`](../../templates/loop-design-template.md).

Each records: the goal and scope, whether the pattern needs a single agent or
a maker/checker split (and why), the orchestration cost if multi-agent, an
explicit L1->L2->L3 progression plan with concrete exit criteria, and the
safety wiring (`gate.yaml`, token budget, circuit breaker thresholds).

Run `npx @cobusgreyling/loop-init` against the design's Goal/Scope/Safety
sections to scaffold the pattern once its design doc is approved.
